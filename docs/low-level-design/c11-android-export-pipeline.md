## `Makefile` Skeleton

```makefile
-include .env
export

GODOT ?= godot
EXPORT_DIR := export
BOOT_CHECK_SECONDS := 60

.PHONY: check apk-debug apk-release clean

check:
	$(GODOT) --headless --path . --import
	$(GODOT) --headless --path . --quit-after $(BOOT_CHECK_SECONDS)

apk-debug: check
	@mkdir -p $(EXPORT_DIR)
	$(GODOT) --headless --export-debug "Android" $(EXPORT_DIR)/syn-grid-debug.apk

apk-release: check
	@test -n "$(KEYSTORE_PATH)" || (echo "KEYSTORE_PATH not set ‚Äî check .env (see .env.example)"; exit 1)
	@test -f "$(KEYSTORE_PATH)" || (echo "Keystore not found at $(KEYSTORE_PATH)"; exit 1)
	@test -n "$(KEYSTORE_PASS)" || (echo "KEYSTORE_PASS not set ‚Äî check .env"; exit 1)
	@test -n "$(KEYSTORE_ALIAS)" || (echo "KEYSTORE_ALIAS not set ‚Äî check .env"; exit 1)
	@mkdir -p $(EXPORT_DIR)
	$(GODOT) --headless --export-release "Android" $(EXPORT_DIR)/syn-grid-release.apk

clean:
	rm -rf $(EXPORT_DIR)
```

Non-negotiable contract points:
- `-include .env` (leading dash) so `check` and `apk-debug` work before a developer has set up release signing.
- `check` is `import` then `quit-after`, never a bare `--quit` ‚Äî see HLD trade-off #2 for why.
- `apk-release` fails closed (non-zero exit, named missing variable) before ever invoking Godot. It must not fall through to producing an APK signed with the wrong identity or an unsigned APK under the release filename.
- `apk-debug` and `apk-release` both depend on `check` so a broken import never silently produces a stale APK.

## `export_presets.cfg.example` Contract

Not literal Godot INI syntax here ‚Äî Cursor generates the real preset via the Godot editor (Project ‚Üí Export) and then commits the resulting `export_presets.cfg` renamed to `.example`, with keystore paths blanked out. The values that must be present in that generated preset:

- Platform: Android
- Package name: `com.nomotomo.syncgrid`
- Min SDK: 24
- Target SDK: 34
- Screen orientation: Portrait
- Permissions: `INTERNET`, `ACCESS_NETWORK_STATE` enabled; no other permissions added without a documented reason
- `keystore/debug` and `keystore/release` fields left blank/placeholder in the `.example` file ‚Äî real paths only ever exist in each developer's own `export_presets.cfg`

## `docs/build.md` Required Outline

Cursor writes the real content once the pipeline is actually verified against a real export; the required sections are:

1. **Prerequisites** ‚Äî exact Godot version (must match `project.godot`'s `config/features`), Android SDK components (build-tools + platform-tools, for `aapt`), export templates installation via Editor ‚Üí Manage Export Templates.
2. **First-time setup** ‚Äî copy `export_presets.cfg.example` ‚Üí `export_presets.cfg`, copy `.env.example` ‚Üí `.env`.
3. **Debug keystore** ‚Äî the exact `keytool` command, note that it's regenerable and low-stakes.
4. **Release keystore** ‚Äî the exact `keytool` command, the mandatory backup-before-first-use warning, explicit statement that this step happens outside any AI-assisted session per project policy.
5. **Running the first build** ‚Äî `make check`, then `make apk-debug`, expected output, where the APK lands.
6. **Size audit** ‚Äî `aapt dump badging export/syn-grid-debug.apk`, how to read the output, the 50MB budget, what to do if over budget (link back to the PRD's gated OGG-conversion decision).
7. **Troubleshooting** ‚Äî at minimum: export template/version mismatch symptoms, missing Android SDK component symptoms, `apk-release` fail-closed messages and what each means.
